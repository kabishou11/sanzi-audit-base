# 分类边界规则

## 缺流程要素 vs 附件内缺少内容

- 缺流程要素：
  强调流程链条没建立、没审批、没决议、没验收、没标准、没台账
- 附件内缺少内容：
  强调材料有壳，但附件中缺少关键内容

判断口诀：

- 缺“流程节点”归缺流程要素
- 缺“附件内容”归附件内缺少内容

## 财务做账不规范 vs 长期挂账未处理

- 财务做账不规范：
  重点在会计处理逻辑错了
- 长期挂账未处理：
  重点在账项长期未清理

判断口诀：

- 账怎么做错了，归财务做账不规范
- 账挂太久没清，归长期挂账未处理

## 合同内容存在问题 vs 流程时间倒置

- 合同内容存在问题：
  合同条款本身不清、不全、不严谨、不合理
- 流程时间倒置：
  合同签订时间、执行时间、验收时间等先后顺序颠倒

判断口诀：

- 看“条款质量”归合同内容存在问题
- 看“时间先后”归流程时间倒置

## 合同内容存在问题 vs 内容一致性问题

- 合同内容存在问题：
  问题出在合同文本本身
- 内容一致性问题：
  问题出在合同、台账、实际、口径之间互相对不上

判断口诀：

- 合同自己有毛病，归合同内容存在问题
- 合同和别的材料对不上，归内容一致性问题

## 业务高度重合 vs 其他问题

- 业务高度重合：
  存在明确重复支付、重复采购、重复计价、服务内容重叠
- 其他问题：
  只是异常，但尚不能明确认定为重合

## 违反相关规定 vs 缺流程要素

- 违反相关规定：
  文本已经上升到明确制度红线、法律风险、政策违规
- 缺流程要素：
  只是程序、审批、附件等不规范，尚未明确触碰红线

判断原则：

- 没有明确规范性结论时，不轻易上升为“违反相关规定”

## 内容一致性问题 vs 业务高度重合

- 内容一致性问题：
  关键信息互相矛盾
- 业务高度重合：
  两笔业务、两类费用、两个服务内容本质重叠

判断原则：

- “对不上”归一致性
- “重复算”归高度重合
