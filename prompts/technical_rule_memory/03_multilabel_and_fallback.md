# 多标签与兜底规则

## 多标签适用场景

以下情况允许输出多标签：

- 一个问题既有流程缺失，又有明显附件缺失
- 一个问题既有合同条款缺陷，又有信息不一致
- 一个问题既有流程时间倒置，又缺流程要素
- 一个问题既有关联交易线索，又存在内容一致性问题

## 多标签排序规则

1. 主问题放前
2. 次问题放后
3. 一般控制在 2 个
4. 最多不超过 3 个

## 主次判断原则

优先考虑：

1. 哪个问题最稳定、最可复用
2. 哪个问题更接近底层技术规则
3. 哪个问题能覆盖更多原文关键信息

## 兜底分类：其他问题

只有在以下情况才使用“其他问题”：

- 文本存在明确异常，但不稳定落入任何主分类
- 问题描述还停留在现象层，未形成稳定规则模式
- 与多个主分类都弱相关，但都不够强

不要把“其他问题”当成默认分类。

## 输出规范

模型输出时应遵守：

- 只输出分类名
- 一行一个分类
- 不解释原因
- 不输出序号
- 不输出 JSON
- 不输出 Markdown
- 优先复用已有稳定分类名

## 分类统一建议

后续尽量统一以下口径：

- “做账不规范”统一到“财务做账不规范”
- “资料缺失”“附件不全”优先落入“附件内缺少内容”或“缺流程要素”
- “重复支付”“重复采购”“服务重叠”优先统一到“业务高度重合”
