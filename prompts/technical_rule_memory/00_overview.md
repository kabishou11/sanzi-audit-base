# 技术规则分类总览

## 任务目标

对单条审计问题，根据 `item_full_title + item_content`，判断其在“技术/规则层面”属于什么异常模式。

这里的分类不是业务领域分类，而是底层规则问题分类。

## 总体原则

1. 优先识别底层规则问题，不停留在业务表面。
2. 优先复用稳定分类名，不频繁新造同义类别。
3. 如果同时命中多个模式，先判断主问题，再决定是否多标签。
4. 如果只是异常现象，但没有稳定落入主分类，可暂归入“其他问题”。
5. “缺流程要素”与“附件内缺少内容”经常同时出现，但要区分主次。
6. “财务做账不规范”与“做账不规范”本质接近，后续尽量统一到“财务做账不规范”。

## 推荐主分类池

优先使用以下分类：

- 缺流程要素
- 附件内缺少内容
- 流程时间倒置
- 长期挂账未处理
- 财务做账不规范
- 合同内容存在问题
- 内容一致性问题
- 业务高度重合
- 违反相关规定
- 关联交易
- 其他问题

## 主干问题结构

从当前示例看，最核心的判断主干是五类：

- 流程缺失
- 时序颠倒
- 挂账久拖不清
- 账务处理错误
- 多来源信息不一致

后续分类时，应优先先检查这五类，再判断是否落入其他专项分类。
